# Social Links Profile - Tasks

Use this as your step-by-step learning roadmap. Check each item only after you complete it yourself.

## 1. Understand The Challenge

- [D] Read [README.md](README.md) fully.
- [D] Read [style-guide.md](style-guide.md) and note the required colors, font, and text size.
- [D] Open the design image and identify all visual blocks:
  - [D] Page background
  - [D] Profile card
  - [D] Avatar image
  - [D] Name
  - [D] Location
  - [D] Bio text
  - [D] Social link buttons

## 2. Prepare Project Files

- [D] Keep your main markup in [index.html](index.html).
- [D] Create a stylesheet file named style.css.
- [D] Link style.css from [index.html](index.html).
- [D] Keep challenge assets paths ready:
  - [D] Avatar: [assets/images/avatar-jessica.jpeg](assets/images/avatar-jessica.jpeg)
  - [D] Favicon: [assets/images/favicon-32x32.png](assets/images/favicon-32x32.png)

## 3. Build HTML Structure First (No CSS Yet)

- [D] Add a top-level main container for the page content.
- [D] Add a card container inside main.
- [D] Add avatar image element with meaningful alt text.
- [D] Add the person name as the main heading.
- [D] Add location text.
- [D] Add bio/quote text.
- [D] Add a list of social links using anchor elements.
- [D] Ensure all links are real anchor tags, not plain text.

## 4. Add CSS Base Styles

- [D] Add a simple reset for default margins/padding.
- [D] Set global box-sizing to border-box.
- [D] Set body font to Inter.
- [D] Set body font size using the guide (14px base).
- [D] Apply background and text colors from [style-guide.md](style-guide.md).

## 5. Layout The Page (Mobile First)

- [D] Start with mobile layout first (small screens).
- [D] Center the card on the page using Flexbox.
- [D] Set sensible card width with max-width.
- [D] Add card padding and rounded corners.
- [D] Add consistent vertical spacing between elements.

## 6. Style Card Content

- [D] Avatar:
  - [D] Set fixed size
  - [D] Make it circular
- [D] Name:
  - [D] Stronger weight
  - [D] Correct color and spacing
- [D] Location:
  - [D] Accent green color
  - [D] Slightly emphasized weight
- [D] Bio:
  - [D] Softer text color
  - [D] Comfortable line-height

## 7. Style Social Buttons

- [D] Make all links look like equal-size buttons.
- [D] Ensure full button width inside the card.
- [D] Add spacing between buttons.
- [D] Add rounded corners and clear text contrast.
- [D] Match normal state colors to the design.

## 8. Add Interaction States (Required)

- [D] Add hover state for each social link button.
- [D] Add focus state for keyboard users.
- [D] Ensure focus state is clearly visible.
- [D] Test with keyboard only:
  - [D] Press Tab to move through links
  - [D] Confirm active focus is easy to see

## 9. Make Desktop Version

- [D] Keep mobile styles as default.
- [D] Add a media query for larger screens.
- [D] Tweak card width/spacing for desktop balance.
- [D] Verify the design still looks good between 320px and large monitors.

## 10. Accessibility And Quality Checks

- [D] Confirm heading structure makes sense.
- [D] Confirm avatar alt text is descriptive.
- [D] Confirm text contrast is readable.
- [D] Confirm all interactive elements can be reached with keyboard.
- [D] Confirm no content overflows at small width.

## 11. Final Compare Against Design

- [D] Compare spacing, sizes, and colors side by side with design.
- [D] Fine-tune only one detail at a time.
- [D] Re-test hover/focus states after every visual change.

## 12. Optional JavaScript Practice (After CSS Is Complete)

- [ ] Add a small script to log which link was clicked.
- [ ] Or generate social links from a small array.
- [ ] Keep it simple and readable.

## 13. Finish And Share

- [D] Update project README using [README-template.md](README-template.md).
- [D] Deploy using GitHub Pages, Netlify, or Vercel.
- [D] Submit solution on Frontend Mentor.
- [D] Ask for feedback on one specific area (for example: spacing or accessibility).
