# Social Links Profile - Tasks

Use this as your step-by-step learning roadmap. Check each item only after you complete it yourself.

## 1. Understand The Challenge

- [D] Read [README.md](README.md) fully.
- [D] Read [style-guide.md](style-guide.md) and note the required colors, font, and text size.
- [D] Open the design image and identify all visual blocks:
  - [D] Page background
  - [D] Profile card
  - [D] Avatar image
  - [D] Name
  - [D] Location
  - [D] Bio text
  - [D] Social link buttons

## 2. Prepare Project Files

- [D] Keep your main markup in [index.html](index.html).
- [D] Create a stylesheet file named style.css.
- [D] Link style.css from [index.html](index.html).
- [D] Keep challenge assets paths ready:
  - [D] Avatar: [assets/images/avatar-jessica.jpeg](assets/images/avatar-jessica.jpeg)
  - [D] Favicon: [assets/images/favicon-32x32.png](assets/images/favicon-32x32.png)

## 3. Build HTML Structure First (No CSS Yet)

- [D] Add a top-level main container for the page content.
- [D] Add a card container inside main.
- [D] Add avatar image element with meaningful alt text.
- [D] Add the person name as the main heading.
- [D] Add location text.
- [D] Add bio/quote text.
- [D] Add a list of social links using anchor elements.
- [D] Ensure all links are real anchor tags, not plain text.

## 4. Add CSS Base Styles

- [ ] Add a simple reset for default margins/padding.
- [ ] Set global box-sizing to border-box.
- [ ] Set body font to Inter.
- [ ] Set body font size using the guide (14px base).
- [ ] Apply background and text colors from [style-guide.md](style-guide.md).

## 5. Layout The Page (Mobile First)

- [ ] Start with mobile layout first (small screens).
- [ ] Center the card on the page using Flexbox.
- [ ] Set sensible card width with max-width.
- [ ] Add card padding and rounded corners.
- [ ] Add consistent vertical spacing between elements.

## 6. Style Card Content

- [ ] Avatar:
  - [ ] Set fixed size
  - [ ] Make it circular
- [ ] Name:
  - [ ] Stronger weight
  - [ ] Correct color and spacing
- [ ] Location:
  - [ ] Accent green color
  - [ ] Slightly emphasized weight
- [ ] Bio:
  - [ ] Softer text color
  - [ ] Comfortable line-height

## 7. Style Social Buttons

- [ ] Make all links look like equal-size buttons.
- [ ] Ensure full button width inside the card.
- [ ] Add spacing between buttons.
- [ ] Add rounded corners and clear text contrast.
- [ ] Match normal state colors to the design.

## 8. Add Interaction States (Required)

- [ ] Add hover state for each social link button.
- [ ] Add focus state for keyboard users.
- [ ] Ensure focus state is clearly visible.
- [ ] Test with keyboard only:
  - [ ] Press Tab to move through links
  - [ ] Confirm active focus is easy to see

## 9. Make Desktop Version

- [ ] Keep mobile styles as default.
- [ ] Add a media query for larger screens.
- [ ] Tweak card width/spacing for desktop balance.
- [ ] Verify the design still looks good between 320px and large monitors.

## 10. Accessibility And Quality Checks

- [ ] Confirm heading structure makes sense.
- [ ] Confirm avatar alt text is descriptive.
- [ ] Confirm text contrast is readable.
- [ ] Confirm all interactive elements can be reached with keyboard.
- [ ] Confirm no content overflows at small width.

## 11. Final Compare Against Design

- [ ] Compare spacing, sizes, and colors side by side with design.
- [ ] Fine-tune only one detail at a time.
- [ ] Re-test hover/focus states after every visual change.

## 12. Optional JavaScript Practice (After CSS Is Complete)

- [ ] Add a small script to log which link was clicked.
- [ ] Or generate social links from a small array.
- [ ] Keep it simple and readable.

## 13. Finish And Share

- [ ] Update project README using [README-template.md](README-template.md).
- [ ] Deploy using GitHub Pages, Netlify, or Vercel.
- [ ] Submit solution on Frontend Mentor.
- [ ] Ask for feedback on one specific area (for example: spacing or accessibility).
